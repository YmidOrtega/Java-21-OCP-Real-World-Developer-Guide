- **Se debe ser capaz de escribir código que utilice operadores de Java.** Este capítulo cubrió una amplia variedad de símbolos de operadores. Se deben repasar varias veces para estar familiarizado con ellos a lo largo del resto del libro.
- **Se debe ser capaz de reconocer qué operadores están asociados con qué tipos de datos.** Algunos operadores pueden aplicarse solo a primitivos numéricos, algunos solo a valores booleanos y algunos solo a objetos. Es importante notar cuándo un operador y un operando(s) no coinciden, ya que es probable que este problema surja en un par de preguntas del examen.
- **Se debe comprender cuándo se requiere _casting_ o cuándo ocurre la promoción numérica.** Siempre que se mezclan operandos de dos tipos de datos diferentes, el compilador necesita decidir cómo manejar el tipo de datos resultante. Cuando se convierte de un tipo de datos más pequeño a uno más grande, la promoción numérica se aplica automáticamente. Cuando se convierte de un tipo de datos más grande a uno más pequeño, se requiere _casting_.
- **Se debe comprender la precedencia de operadores de Java.** La mayoría de los operadores de Java con los que se trabajará son binarios, pero el número de expresiones es a menudo mayor que dos. Por lo tanto, se debe entender el orden en el que Java evaluará cada símbolo de operador.
- **Se debe ser capaz de escribir código que utilice paréntesis para anular la precedencia de operadores.** Se pueden utilizar paréntesis en el código para cambiar manualmente el orden de precedencia.
