
- **Se debe ser capaz de escribir código utilizando un método `main()`.** Un método `main()` se escribe habitualmente como `public static void main(String[] args)`. Los argumentos se referencian comenzando con `args[0]`. El acceso a un argumento que no fue pasado provocará que el código lance una excepción.
- **Se debe comprender el efecto del uso de paquetes e importaciones.** Los paquetes contienen clases Java. Las clases pueden importarse por nombre de clase o mediante comodín (_wildcard_). Los comodines no buscan en subdirectorios. En caso de conflicto, las importaciones por nombre de clase tienen prioridad. Las declaraciones de paquete e importación son opcionales. Si están presentes, ambas se colocan antes de la declaración de la clase en ese orden.
- **Se debe ser capaz de reconocer un constructor.** Un constructor tiene el mismo nombre que la clase. Se asemeja a un método sin tipo de retorno.
- **Se debe ser capaz de identificar declaraciones e inicializaciones legales e ilegales.** Se pueden declarar e inicializar múltiples variables en la misma sentencia cuando comparten un tipo. Las variables locales requieren una inicialización explícita; otras utilizan el valor predeterminado para ese tipo. Los identificadores pueden contener letras, números, símbolos de moneda o `_`, aunque no pueden comenzar con números. Además, no es posible definir un identificador que sea solo un carácter de guion bajo simple. Los literales numéricos pueden contener guiones bajos entre dos dígitos, como `1_000`, pero no en otros lugares, como `_100_.0_`.
- **Se debe comprender cómo crear bloques de texto.** Un bloque de texto comienza con `"""` en la primera línea. En la línea siguiente comienza el contenido. La última línea termina con `"""`. Si las `"""` finales están en su propia línea, se incluye un salto de línea final.
- **Se debe ser capaz de utilizar `var` correctamente.** `var` se utiliza para una variable local. Una variable `var` se inicializa en la misma línea donde se declara y, si bien puede cambiar de valor, no puede cambiar de tipo. Una variable `var` no puede inicializarse con un valor `null` sin un tipo, ni puede utilizarse en declaraciones de múltiples variables.
- **Se debe ser capaz de determinar dónde entran y salen del alcance las variables.** Todas las variables entran en alcance cuando se declaran. Las variables locales salen del alcance cuando termina el bloque en el que se declaran. Las variables de instancia salen del alcance cuando el objeto es elegible para la recolección de basura. Las variables de clase permanecen en alcance mientras el programa se esté ejecutando.
- **Se debe saber identificar cuándo un objeto es elegible para la recolección de basura.** Se recomienda dibujar un diagrama para realizar el seguimiento de referencias y objetos al rastrear el código. Cuando no hay flechas apuntando a una caja (objeto), este es elegible para la recolección de basura.
